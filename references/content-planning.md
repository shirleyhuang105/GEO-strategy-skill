# 内容规划指南

## 内容支柱策略

### 什么是内容支柱

内容支柱（Content Pillar）是围绕核心主题构建的内容体系：

```
           [支柱页面]
              │
    ┌─────────┼─────────┐
    │         │         │
[子内容1] [子内容2] [子内容3]
```

- **支柱页面**：全面介绍核心主题的综合性内容
- **子内容**：深入探讨具体细分话题

### 支柱选择标准

一个好的内容支柱应该：
- 与产品/服务高度相关
- 有足够的话题延展空间
- 目标客户关心的领域
- 有搜索需求支撑

### 支柱规划示例

假设产品是「企业数据分析平台」：

**支柱1：数据分析基础**
- 什么是数据分析
- 数据分析方法论
- 数据分析工具选型
- 数据分析最佳实践

**支柱2：业务场景应用**
- 营销数据分析
- 销售数据分析
- 运营数据分析
- 财务数据分析

**支柱3：技术实践指南**
- 数据采集方案
- 数据建模方法
- 报表设计原则
- 数据可视化技巧

**支柱4：行业解决方案**
- 零售行业数据分析
- 金融行业数据分析
- 制造行业数据分析
- 互联网行业数据分析

## 买家旅程内容映射

### 认知阶段 (Awareness)

用户状态：意识到问题，开始寻找信息

内容目标：教育、建立认知

适合的内容类型：
- 概念解读文章
- 趋势分析报告
- 行业洞察
- 问题诊断指南

关键词特征：
- "什么是..."
- "为什么..."
- "...趋势"
- "...挑战"

### 考虑阶段 (Consideration)

用户状态：了解问题，评估解决方案

内容目标：展示价值、建立信任

适合的内容类型：
- 解决方案介绍
- 产品对比分析
- 选型指南
- 最佳实践指南
- 客户案例（初步）

关键词特征：
- "...解决方案"
- "...vs..."
- "如何选择..."
- "...最佳实践"

### 决策阶段 (Decision)

用户状态：准备选择供应商

内容目标：促进转化

适合的内容类型：
- 详细案例研究
- ROI 计算器
- 实施指南
- 定价说明
- 免费试用/演示

关键词特征：
- "...价格"
- "...试用"
- "...实施"
- "...ROI"

## 内容优先级排序

### 优先级评估维度

**业务价值 (权重 40%)**
- 与产品核心价值的关联度
- 对转化的潜在贡献
- 是否触及决策者

**搜索机会 (权重 30%)**
- 搜索量/需求
- 竞争难度
- 排名可行性

**资源投入 (权重 30%)**
- 内容创作难度
- 所需时间
- 专业知识要求

### 优先级矩阵

| 优先级 | 特征 | 建议 |
|-------|------|------|
| P0 | 高业务价值 + 高搜索机会 + 低投入 | 立即执行 |
| P1 | 高业务价值 + 中等搜索机会 | 优先执行 |
| P2 | 中等业务价值 + 高搜索机会 | 计划执行 |
| P3 | 其他组合 | 后续考虑 |

## 内容日历规划

### 发布节奏建议

对于 B2B 企业：
- 建议频率：每周 1-2 篇高质量内容
- 避免堆量：质量优先于数量

### 内容类型配比

建议比例：
- 认知阶段内容：40%
- 考虑阶段内容：35%
- 决策阶段内容：25%

### 主题轮换

确保内容支柱间的平衡：
- 每个支柱每月至少 1 篇内容
- 避免单一支柱内容过于集中

## 内链策略

### 内链原则

- 支柱页面链接到所有相关子内容
- 子内容链接回支柱页面
- 相关子内容之间互相链接
- 锚文本使用目标关键词

### 内链结构示例

```
支柱页：数据分析完全指南
  ├── 链接到：数据分析方法论
  ├── 链接到：数据分析工具选型
  └── 链接到：数据分析最佳实践

子页：数据分析方法论
  ├── 返回链接：数据分析完全指南
  ├── 相关链接：数据分析工具选型
  └── 相关链接：数据分析案例
```
